Spike 3 — Can the browser reliably capture Ithraa's Alt shortcuts?

Target combos: Alt+X Alt+Z Alt+Q Alt+W Alt+M Alt+0 Alt+9 Alt+K Alt+L. The listener below is attached to the contenteditable box and matches on event.code (physical key position: KeyX/KeyZ/KeyQ/KeyW/KeyM/Digit0/Digit9/KeyK/KeyL) combined with event.altKey, then calls preventDefault(). This mirrors the PLAN.md §5 strategy ("event.code-based so macOS Option works").

Caveat (read before trusting automated results): synthetic keyboard events sent via browser-automation protocols (CDP / WebDriver "key" actions) do not necessarily reproduce everything a physical macOS Option-key press does at the OS/IME level (e.g. dead-key composition, the OS swallowing the keystroke before it reaches the browser, or differences between how CDP synthesizes altKey vs a real hardware event). A PASS/FAIL from this automated run is evidence, not proof, for real hardware. This page is built to remain usable for a human to test manually — focus the box below and physically press each Option/Alt combo on a real Mac keyboard, then read the log table.
